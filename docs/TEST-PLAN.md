## Affected Pages/Routes
- `/` (라인메이트 대시보드) — 본인 데이터만 표시 확인
- `/submit` (참여 입력) — 중복 제출 방지, finalized 달 차단
- `/admin/approvals` (승인 큐) — 승인·거절·단가 보정 동작
- `/admin/settlement/[year_month]` (월 마감) — finalize RPC 정확성
- `/admin/export/[year_month]` (엑셀 출력) — 기존 양식 셀 1:1 일치

## Key Interactions to Verify
1. 라인메이트가 참여 제출 → 관리자에게 이메일 알림 발송
2. 관리자가 승인 → 라인메이트에게 이메일 알림 발송
3. "마감 확정" 버튼 클릭 → 해당 월 수정 잠금 확인
4. 이미 finalized된 달에 참여 내역 추가 시도 → 오류 반환

## Edge Cases
- 같은 날·같은 프로젝트 중복 제출 → 서버에서 거부
- unit_price NULL인 참여 건 → project.default_unit_price 자동 적용
- 소수점 원화 계산 → 원 단위 반올림
- 0건 참여 라인메이트가 포함된 달 엑셀 출력 → 빈 행 정상 처리
- pending 상태 라인메이트가 API 호출 → 401/403 반환
- 라인메이트 A가 라인메이트 B의 /participations?linemate_id=B 호출 → RLS가 빈 결과 반환

## Critical Paths
1. 참여 입력 → 관리자 승인 → 월 정산 집계 → 엑셀 출력: 금액 일치 검증
2. finalize_month RPC: status=draft 확인 → total_amount 계산 → finalized 저장 (원자적 실행)
3. 엑셀 파일 스냅샷: 기존 "(주)라인교육연구소_2026년 ○월 인건비내역_김연미.xlsx" 양식과 셀 위치·수식 1:1 일치

## Test Types Required
- 단위 테스트 (Vitest): 정산 계산 함수, unit_price fallback 로직
- 통합 테스트 (Supabase Test Helpers): RLS 정책, finalize_month RPC
- 스냅샷 테스트 (ExcelJS): 엑셀 출력 파일 바이트 비교
- E2E 테스트 (Playwright, Week 4): 전체 제출→승인→정산→다운로드 플로우
